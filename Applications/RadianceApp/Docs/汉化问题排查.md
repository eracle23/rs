# 汉化不生效问题排查

重新生成 `SlicerTranslations_zh_CN` 后仍无汉化，可按以下顺序排查。

---

## 六、翻译失败根本原因分析（技术细节）

### 原因 1：强制中文补丁未触发（已修复）

**现象**：即使有 zh_CN 的 .qm 文件，界面仍为英文。

**根因**：`slicer-force-zh-cn-for-vme.patch` 原先检查 `applicationName().contains("VisionMagicEcosystem")`，但 `Main.cxx` 中设置了 `setApplicationName("VisionMagic")`，导致条件永远不满足，补丁逻辑从未执行。

**修复**：补丁已改为检查 `applicationName().contains("VisionMagic")`，可同时匹配 VisionMagic 与 VisionMagicEcosystem。

### 原因 2：首次运行时用户设置为空

**现象**：新安装或首次启动时界面为英文。

**根因**：Slicer 的 `init()` 仅当 `userSettings()->value("Internationalization/Enabled").toBool()` 为 true 时才加载翻译。首次运行时用户设置尚未写入，该值为 false，`loadLanguage()` 不会被调用。

**依赖**：强制中文补丁在上述情况下将 `ApplicationLocaleName` 设为 zh_CN 并调用 `loadLanguage()`，从而覆盖此行为。

### 原因 3：DefaultSettings 可能未被加载

**现象**：DefaultSettings.ini 中的 language=zh_CN 未生效。

**根因**：`slicerDefaultSettingsFilePath()` 使用 `applicationName() + "DefaultSettings.ini"`，即 `VisionMagicDefaultSettings.ini`；而 CMake 安装的文件名为 `VisionMagicEcosystemDefaultSettings.ini`，路径不匹配时 `defaultSettings()` 返回 nullptr，默认配置不会合并到用户设置。

**影响**：首次运行时无法从 DefaultSettings 获取 language=zh_CN，需依赖强制中文补丁。

### 原因 4：.qm 文件路径或 Slicer_QM_OUTPUT_DIRS

**根因**：`translationFolders()` 依次查找：`toSlicerHomeAbsolutePath(Slicer_QM_DIR)`、`Slicer_QM_OUTPUT_DIRS` 中的路径、以及可执行文件目录及 `translations` 子目录。若这些路径中均无 `*zh_CN.qm` 或 `*zh-CN.qm`，则翻译不会加载。

**排查**：确认 `{构建目录}\Slicer-build\bin\translations\` 下存在 `Slicer_zh-CN.qm`、`CTK_zh-CN.qm`、`LanguagePacks_zh-CN.qm`。

### 原因 5：扩展模块无中文翻译

**现象**：部分模块（如 TotalSegmentator、MarkupsToModel）仍为英文。

**根因**：扩展来自第三方仓库，其 `parent.title` 等字符串为英文且未包含在 Slicer 官方 .ts 中。需在 `LanguagePacks_zh-CN.ts` 中增加对应条目，或由扩展自身提供翻译。

---

## 一、强制中文补丁（推荐优先尝试）

已新增 `slicer-force-zh-cn-for-vme.patch`，在应用名为 VisionMagic/VisionMagicEcosystem 且当前为 en_US 时，**强制启用 zh_CN**，避免受 DefaultSettings / 用户设置影响。

### 应用步骤

1. **删除 patch 标记**（让所有补丁重新应用，包括新补丁）：
   ```bat
   del /q "E:\build\_deps\slicersources-src\.rs_patches_applied" 2>nul
   ```
   若 Slicer 源码在其它目录，删除对应 `slicersources` 目录下的 `.rs_patches_applied`。

2. **重新配置**：
   ```bat
   cd /d E:\build
   cmake E:\GitHub\rs0\rs
   ```

3. **重新生成**（需要重新编译 Slicer Base，因为修改了 qSlicerCoreApplication.cxx）：
   - 在 VS 中：**生成 → 重新生成解决方案**
   - 或命令行：`cmake --build E:\build --config RelWithDebInfo -- /m`

4. **清除用户设置**（避免旧设置干扰）：
   ```bat
   rmdir /s /q "%APPDATA%\VisionMagic" 2>nul
   rmdir /s /q "%LOCALAPPDATA%\VisionMagic" 2>nul
   del /q "%APPDATA%\Vision Magic Ecosystem.com" 2>nul
   ```

5. 启动应用测试。

## 二、确认应用内语言设置

Slicer 只有在满足以下两项时才会加载翻译：

1. **国际化和本地化**：`Internationalization/Enabled` = true  
2. **语言**：`language` = zh_CN

### 检查步骤

1. 启动应用 → **Edit → Application Settings**
2. 在 **General** 中确认 **Language** 为 **中文** 或 **zh_CN**
3. 在 **Internationalization** 中确认已勾选 **Enable Internationalization**
4. 点击 OK，**完全退出应用后再重新启动**

---

## 三、确认 .qm 是否生成成功

### 3.1 构建树中 .qm 所在位置

生成路径通常为：

```
{构建目录}\Slicer-build\bin\translations\
```

应能看到：`Slicer_zh-CN.qm`、`CTK_zh-CN.qm`、`LanguagePacks_zh-CN.qm`

### 3.2 若没有 .qm 文件

1. 检查 CMake 输出是否有：`Building SlicerLanguageTranslations for zh-CN...`
2. 确认 `Slicer_BUILD_I18N_SUPPORT` 已开启
3. 在 VS 中右键 `SlicerTranslations_zh_CN` → 重新生成

---

## 四、用户设置存储位置

Slicer 可能将用户设置存储在以下位置（取决于 org/app 配置）：

- `%APPDATA%\VisionMagic\`
- `%LOCALAPPDATA%\VisionMagic\`
- `%APPDATA%\Vision Magic Ecosystem.com\`

清除汉化相关配置时，建议一并检查并删除以上目录中的 `.ini` 文件。

---

## 五、快速检查清单

| 检查项 | 操作 |
|--------|------|
| 强制中文补丁 | 删除 `.rs_patches_applied` → 重新配置 → 重新生成**整个解决方案** |
| 用户设置 | 删除上述 APPDATA/LOCALAPPDATA 目录 |
| 语言设置 | Settings → General → Language = 中文 |
| 国际化 | Settings → Internationalization → 已勾选 |
| 重启应用 | 修改设置后必须完全退出再启动 |
