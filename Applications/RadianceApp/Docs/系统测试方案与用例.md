## 2. 功能模块梳理

### 2.1 应用级功能（RadianceApp 定制）

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 启动与初始化 | 启动画面、用户数据库初始化、日志初始化 | 必须成功方能进入主界面 |
| 用户认证 | 登录、退出登录、记住登录状态 | 首次启动需登录 |
| 用户管理 | 修改密码、用户管理（admin）、当前用户显示 | 用户菜单 |
| 数据导入校验 | VolumeImportValidator | 仅允许 DICOM；尺寸/角度/阈值限制 |
| 医生批注 | DoctorAnnotationWidget | 随 Volume 保存的文本批注 |
| 界面定制 | 工作区/帮助/用户菜单、主题、工具栏图标 | 基于 Slicer 的定制 |
| 日志与帮助 | 打开日志目录、打开当前日志 | 帮助菜单 |

### 2.2 Slicer 核心模块

| 模块 | 功能 | 默认常用 |
|------|------|----------|
| Data | 数据加载/导入 | ✓ |
| DICOM | DICOM 浏览与导入 | ✓（Home） |
| Volumes | 体数据查看与处理 | ✓ |
| VolumeRendering | 体绘制 | ✓ |
| SegmentEditor | 分割编辑 | ✓ |
| Transforms | 变换 | ✓ |
| Markups | 标记点 | ✓ |
| Models | 模型 | ✓ |
| SlicerElastix | 配准（Elastix） | ✓ |
| CompareVolumes | 体积比较 | 内置 |
| LandmarkRegistration | 地标配准 | 内置 |

### 2.3 扩展模块

| 扩展 | 功能 |
|------|------|
| TotalSegmentator | 全自动解剖分割 |
| SlicerNNUnet | nnUNet 分割 |
| SegmentEditorExtraEffects | 分割编辑扩展效果 |
| SlicerAirwaySegmentation | 气道分割 |
| MarkupsToModel | 标记点生成模型 |

## 3. 系统测试方案

### 3.1 测试环境

- **操作系统**：Windows 10/11（64 位）
- **硬件**：满足 Slicer 推荐配置（GPU 建议用于体绘制、分割）
- **测试数据**：标准 DICOM 体数据（CT/MRI）、无效格式样本

### 3.2 测试策略

- **功能测试**：按模块逐项验证主要流程
- **边界测试**：导入校验的边界值（尺寸超限、格式非法、阈值超限）
- **角色测试**：普通用户 vs 管理员（用户管理可见性）
- **回归测试**：关键路径在每次发布前执行

### 3.3 优先级定义

- **P0**：阻塞性，影响核心使用
- **P1**：重要功能，影响典型工作流
- **P2**：次要功能，可接受临时替代方案

## 5. 测试执行建议

### 5.1 冒烟测试（每次构建后）

执行：ST-001、ST-003、ST-004、ST-020、ST-040、ST-041、ST-060、ST-061、ST-090

### 5.2 回归测试（发布前）

- 执行所有 P0、P1 用例
- 根据变更模块补充 P2 用例

### 5.3 测试数据准备

- **有效 DICOM**：标准 CT 头部/胸部序列
- **无效格式**：.nii、.nrrd、.mha 等非 DICOM
- **边界数据**：超大尺寸、超阈值范围（若可构造）
- **多用户**：admin 与普通用户各一

### 5.4 缺陷管理

- 缺陷等级与用例优先级对应
- 记录：步骤、环境、预期/实际、截图或日志
- P0 缺陷需在发布前修复或评估风险
